## When to use this skill

- When creating or editing React components that need to be accessible
- When implementing keyboard navigation and visible focus indicators
- When using semantic HTML elements (nav, main, button, article, etc.)
- When adding ARIA attributes to enhance complex components
- When ensuring color contrast ratios meet WCAG standards (4.5:1 for normal text)
- When providing alternative text for images and meaningful labels for form inputs
- When testing components with screen readers (VoiceOver, NVDA, JAWS)
- When implementing proper heading hierarchy (h1-h6) for document structure
- When managing focus in dynamic content, modals, and single-page navigation
- When ensuring all interactive elements are keyboard accessible
- When avoiding reliance on color alone to convey information
- When working with shadcn/ui components to maintain their accessibility features

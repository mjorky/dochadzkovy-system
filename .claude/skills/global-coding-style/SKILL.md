## When to use this skill

- When writing TypeScript code in both backend (NestJS) and frontend (Next.js)
- When naming variables, functions, classes, and files consistently
- When ensuring consistent code formatting with Prettier and ESLint
- When writing meaningful, descriptive names that reveal intent
- When creating small, focused functions that do one thing well
- When removing dead code, commented-out blocks, and unused imports
- When ensuring consistent indentation and code structure
- When applying DRY (Don't Repeat Yourself) principle to extract common logic
- When refactoring code to improve readability without changing behavior
- When deciding whether backward compatibility is needed (assume not unless specified)
- When working with TypeScript files (.ts, .tsx) across the entire stack
- When ensuring code passes linting and formatting checks

When implementing feature flags for incomplete features. When maintaining documentation like README files. When following the established code review process.

## When to use this skill

- When organizing or restructuring files and directories in the project
- When writing git commit messages following the project's conventions
- When creating feature branches and pull requests with clear descriptions
- When managing environment variables and ensuring secrets aren't committed
- When adding, updating, or documenting project dependencies
- When participating in code reviews as author or reviewer
- When determining test coverage requirements before merging
- When implementing feature flags for incomplete or experimental features
- When maintaining project documentation (README, setup instructions, architecture docs)
- When updating changelogs or release notes
- When setting up new environments or configuring deployment pipelines
- When working with monorepo structures or multi-package projects

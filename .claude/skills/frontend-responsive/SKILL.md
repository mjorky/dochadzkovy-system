## When to use this skill

- When creating or editing components that need to work across multiple screen sizes
- When using TailwindCSS responsive modifiers (sm:, md:, lg:, xl:, 2xl:)
- When implementing mobile-first design (start with mobile layout, enhance for larger screens)
- When using Tailwind's standard breakpoints consistently across the application
- When creating fluid layouts with percentage-based widths and flexible containers
- When using relative units (rem, em) instead of fixed pixels for better scalability
- When ensuring tap targets are at least 44x44px for mobile touch interactions
- When optimizing images and assets for mobile network conditions
- When maintaining readable typography across all breakpoints
- When prioritizing content for smaller screens through thoughtful layout decisions
- When testing UI changes on mobile, tablet, and desktop screen sizes
- When implementing responsive navigation patterns (hamburger menus, drawer navigation)
